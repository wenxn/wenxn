1.了解cpu，内存，硬盘等硬件设施

cpu的信息通过 cat /proc/cpuinfo可进行查看
grep “physical id” | sort | uniq | wc -l 查看物理cpu个数
grep “processor” | wc -l 查看逻辑cpu个数
grep “cpu cores “ | uniq 查看cpu核数

一般：逻辑cpu的个数=核心数的两倍
超线程：逻辑数=核心数的两倍（x86）

cache和buffet区别：
1.cache属于高速缓存，cpu从内存中直接读取数据，减少cpu等待（文件系统的内存）
2.buffet缓冲区，存储速度不同步的设备之间的传输，进程之间等待少（块设备的读写缓冲区）
监控内存使用较高的前五位进程
ps -aux | sort -r -k 4 | head -6

硬盘的文件系统（Linux主要以ext系列为主）
硬盘区别：
1.SSD固态硬盘，日常读写储存快，成本高
2.HHD机械硬盘，成本低，适合做大容量存储

fdisk -l 查看物理硬盘列表

网络：
ifstat，iftop 进行网络接口监测，实时流量监控

3.进行socket通信实践–python的多人聊天室

了解模块
socket模块是针对服务端和客户端socket进行通信 、

服务器端：

1.创建socket对象，如socket.AF_INET,SOCK_STREAM)
2.绑定ip地址和端口号，bind(adress,port)
3.接受请求，接受指定的用户数listen(num)
4.等待用户请求连接 accept() = con,addr，返回socket对象及对应地址
5.处理阶段，传输字节数，recv(num)，send(num)
6.关闭连接，close()

客户端：

1.创建socket对象，如socket.AF_INET,SOCK_STREAM)
2.连接服务器，connect()方法
3.处理阶段，传输字节数，recv(num)，send(num)
4.关闭连接，close()

线程中最简单的同步就是锁机制–threading模块为低级线程控制。

多线程使用threading模块，condition实现复杂同步。为生产者与消费者模型，但在python中只能交替进行，并不能做到真正的并发执行

conn=threading.Condition()创建一个对象，使用对象可以在某些事件触发或者达到特定条件后才处理数据
wait()：线程挂起，直到收到一个notify通知才会被唤醒继续进行
notify()：通知其他线程，那些被挂起的线程接到这个通知后会执行
notifyAll()：wait状态线程多，通知所有线程
acquire()：获取锁
release():释放锁

多线程缺点：很难保证线程间资源共享工作正常，效率低下

python管理异步socket的API有三种：select，poll，epoll
select函数是对底层操作系统直接访问的接口，可用来监控sockets，files，pipes，等待io完成

select模块
select.select(rlist,wlist,xlist[timeout])
传递三个参数，
一个为输入而观察的文件对象列表可用来监听多个socket连接的可读事件，分为两部分，server socket监听客户端请求，accept（）方法加入rlist，client socket为客户端发送数据，调用recv（）方法获取，检查用户的输入
一个为输出
一个为错误，结果返回三个tuple

select模块中，默认为阻塞模式，如果设置为非阻塞conn.setblocking(0)

epoll有两种模式
1.边沿触发：在读取/写入事件发生时只返回一次，后续再处理所有
2.状态触发：重复调用只返回重复事件

git操作

2.git add file 将本地文件放在存储区
3.git remote add origin git@github.com:xxx/xxx 添加远程仓库
4.git push -u origin 推送到远程仓库

8月遇到的问题，原因以及对应的解决方法

1.python2运行时，出现“Non-ASCII character ‘xe5’ in file”的报错问题。

原因：python使用默认的ASCII作为编码方式
解决方法:
只需在开头加上

#-*-coding：UTF-8 -*-

2.global name is not define

原因：全局变量未设定好
解决方法：查看函数引用时是否传入self

3.git更新被拒绝，因为远程仓库包含本地尚不存在的提交

解决方法：git push -u origin+master
